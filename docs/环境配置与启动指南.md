# 环境配置与启动指南

本文档用于本项目本地开发环境的初始化与日常启动，覆盖以下内容：

- Node.js / npm、MySQL、本地区块链的准备
- 后端 `.env` 配置
- 数据库初始化 SQL 执行顺序
- 前端、后端、区块链的手动启动命令
- 一键启动脚本 `startall.ps1` / `startall.bat` 的用法

## 1. 项目结构与本地端口

本地开发时主要使用以下三个子系统：

- `frontend`：Vue 3 + Vite 前端，默认端口 `5173`
- `backend`：Node.js + Express 后端，默认端口 `3000`
- `backend/hardhat-local`：Hardhat 本地区块链，默认 RPC `http://127.0.0.1:18545`

默认访问地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:3000`
- 本地链 RPC：`http://127.0.0.1:18545`

前端开发环境默认会直接请求 `http://127.0.0.1:3000`，不再依赖 Vite 的 `/api` 代理转发；如需覆盖，可自行设置 `VITE_API_BASE_URL`。

## 2. 前置依赖

建议本机具备以下环境：

- Node.js 18+，并能正常执行 `npm`
- MySQL 8.x
- Windows PowerShell 或 PowerShell 7

首次拉代码后，需要分别安装前后端依赖：

```powershell
cd backend
npm install

cd ../frontend
npm install
```

说明：

- `backend/node_modules` 负责后端和 Hardhat 本地链脚本依赖
- `frontend/node_modules` 负责 Vue/Vite 前端依赖

## 3. 后端环境变量配置

后端配置模板在 `backend/.env.example`，本地启动前请复制为 `backend/.env`。

示例：

```env
PORT=3000
NODE_ENV=development
JWT_SECRET=change-this-in-production
JWT_EXPIRES_IN=7d
NONCE_TTL_MS=300000
ALLOW_DEV_AUTO_REGISTER=false
ADMIN_BOOTSTRAP_KEY=change-this-bootstrap-key
ACTIVATION_CODE_TTL_MINUTES=60
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=aviation_maintenance
DB_USER=root
DB_PASSWORD=change-me
DB_CONNECTION_LIMIT=10
```

至少要确认以下字段正确：

- `PORT`：后端监听端口，默认 `3000`
- `JWT_SECRET`：JWT 密钥，本地开发也不要留空
- `ADMIN_BOOTSTRAP_KEY`：管理员预注册接口需要的启动密钥
- `DB_HOST` / `DB_PORT` / `DB_NAME` / `DB_USER` / `DB_PASSWORD`：MySQL 连接信息

后端启动时会先测试数据库连接，如果数据库没启动、库名不存在或密码错误，服务会直接启动失败。

## 4. 数据库初始化

本项目数据库初始化脚本位于 `backend/sql`：

- `init_auth.sql`：认证、用户、角色、权限相关表
- `init_maintenance_v2.sql`：检修记录、签名、附件 manifest 等业务表
- `review_user_permissions.sql`：辅助检查用户权限的视图和查询语句

建议先手动创建数据库：

```sql
CREATE DATABASE aviation_maintenance CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后按顺序导入：

```powershell
mysql -h 127.0.0.1 -P 3306 -u root -p aviation_maintenance < backend/sql/init_auth.sql
mysql -h 127.0.0.1 -P 3306 -u root -p aviation_maintenance < backend/sql/init_maintenance_v2.sql
```

如果需要审查当前用户角色与权限，可以再执行：

```powershell
mysql -h 127.0.0.1 -P 3306 -u root -p aviation_maintenance < backend/sql/review_user_permissions.sql
```

注意：

- `init_auth.sql` 和 `init_maintenance_v2.sql` 都是开发用重建脚本，包含 `DROP TABLE IF EXISTS`
- 不要在生产环境直接执行这两个脚本

## 5. 手动启动方式

### 5.1 启动本地区块链

在项目根目录执行：

```powershell
cd backend
npm run chain:node
```

对应实际命令为：

```powershell
npm --prefix hardhat-local run node
```

本地链默认信息：

- RPC：`http://127.0.0.1:18545`
- Chain ID：`31337`

### 5.2 编译并部署本地合约

仅启动 Hardhat 节点还不够，后端如果要真正调用 maintenance V2 合约，还需要先完成编译和部署。

在项目根目录执行：

```powershell
cd backend
npm run chain:compile
npm run chain:deploy:v2
```

部署成功后，会在以下位置写入本地部署信息：

- `backend/hardhat-local/deployments/local.json`

如果后端报下面这类错误：

```text
未找到 maintenance V2 合约部署信息，请先执行 npm run chain:deploy:v2
```

说明链已经起来了，但合约还没有部署到本地链。

### 5.3 启动后端

```powershell
cd backend
npm run dev
```

启动成功后，终端会输出类似：

```text
Backend server listening on http://127.0.0.1:3000
```

### 5.4 启动前端

```powershell
cd frontend
npm run dev
```

前端默认通过 Vite 启动在 `5173` 端口。

当前前端接口基地址规则如下：

- 若设置了 `VITE_API_BASE_URL`，优先使用该值
- 若未设置且处于开发环境，则默认直接访问 `http://127.0.0.1:3000`
- 若为生产构建，则继续使用相对路径

## 6. 一键启动脚本

项目根目录已经提供：

- `startall.ps1`
- `startall.bat`

用途：

- 打开 3 个独立的 `cmd` 窗口
- 分别启动 Hardhat 本地链、后端 API、前端 Vite

### 6.1 推荐用法

直接双击：

```text
startall.bat
```

或者在 PowerShell 中执行：

```powershell
.\startall.ps1
```

### 6.2 Dry Run

如果只想查看它会执行哪些命令，不真正打开窗口：

```powershell
.\startall.ps1 -DryRun
```

### 6.3 脚本当前行为

脚本当前行为如下：

- 启动 Hardhat 本地链窗口
- 等待本地 RPC `http://127.0.0.1:18545` 就绪
- 自动执行一次 `npm run chain:deploy:v2`
- 自动执行一次 `npm run db:sync:role-permissions`
- 自动执行一次 `npm run db:seed:demo-records`
- 再启动后端 API 窗口
- 等待后端 `http://127.0.0.1:3000/health` 就绪
- 最后启动前端 Vite 窗口
- 等待前端 `http://127.0.0.1:5173/` 就绪

`db:seed:demo-records` 会在每次本地测试链启动时自动清空并重建 maintenance 测试数据，生成一批可直接在前端查看的演示记录。

自动注入的记录状态包括：

- `submitted`
- `rejected`
- `released`
- `rii_approved`

它仍然不会自动执行以下动作：

- 不会自动创建数据库
- 不会自动导入 SQL
- 不会自动执行 `npm install`

因此第一次启动本地环境时，仍然建议先手动完成：

1. 安装依赖
2. 配置 `backend/.env`
3. 初始化 MySQL
4. 至少先执行一次 `npm run chain:compile`
5. 再使用一键启动脚本做日常开发启动

## 7. 推荐启动顺序

如果是第一次启动本地环境，推荐顺序如下：

1. 启动 MySQL
2. 创建数据库 `aviation_maintenance`
3. 导入 `init_auth.sql`
4. 导入 `init_maintenance_v2.sql`
5. 启动 Hardhat 本地链
6. 执行 `npm run chain:compile`
7. 使用 `startall.bat` 或 `startall.ps1` 启动本地链、自动部署合约、启动后端和前端

如果数据库已经准备好，日常开发通常只需要：

1. 启动 MySQL
2. 双击 `startall.bat`

## 9. 启动后快速验证

可以按下面方式快速检查环境是否正常：

### 9.1 检查前端

浏览器打开：

```text
http://127.0.0.1:5173
```

如果页面能打开，说明 Vite 前端正常。

### 9.2 检查后端

确认前端页面里的认证流程可以正常调用 `/api/auth/*`。

若前端能打开但认证接口报错，优先检查：

- 后端是否真的监听在 `3000`
- `backend/.env` 是否存在
- MySQL 是否可连

### 9.3 检查本地链与合约

提交 maintenance 记录时，如果后端返回找不到部署信息或链上调用失败，优先检查：

- Hardhat 本地链窗口是否还在运行
- 是否执行过 `npm run chain:deploy:v2`
- `backend/hardhat-local/deployments/local.json` 是否存在

## 10. 常见问题

### 10.1 后端启动失败

常见原因：

- `backend/.env` 不存在
- MySQL 没启动
- 数据库名不存在
- `DB_USER` / `DB_PASSWORD` 不正确

### 10.2 前端接口调用失败

常见原因：

- 后端没有启动
- 后端端口不是 `3000`
- `VITE_API_BASE_URL` 配置错误
- 浏览器里保留了旧登录态或旧页面缓存，建议刷新后重新登录

### 10.3 链相关接口失败

常见原因：

- Hardhat 本地链未启动
- 合约未部署
- 合约产物不存在，此时先执行 `npm run chain:compile`

### 10.4 双击 bat 没反应

常见原因：

- 本机未安装 Node.js / npm
- PowerShell 执行环境受限
- 路径不在项目根目录

建议先在 PowerShell 手动执行：

```powershell
.\startall.ps1 -DryRun
```

确认脚本打印出的三条命令都正常后，再直接双击 bat。
